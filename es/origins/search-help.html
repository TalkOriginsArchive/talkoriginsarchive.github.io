Consejos para buscar

[Esto está adaptado de las instrucciones del lenguaje de consultas para Microsoft Index Server]

En lo más simple, una consulta puede ser solo una palabra o una frase. Pero con los consejos de esta página, puede ampliar el alcance de su consulta para obtener resultados más completos. Para consultas más complejas y más ejemplos, consulte la página de Lenguaje de consulta.

  • Busque más de una palabra. Utilice los operadores booleanos AND y OR para conectar varias palabras en su consulta. Por ejemplo, escriba dog and cat para encontrar solo los documentos que contengan ambas palabras, dog y cat. Escriba dog or cat para encontrar todos los documentos que contengan alguna de las dos palabras. No escriba dog cat a menos que busque instancias en las que esas dos palabras aparezcan directamente una junto a la otra.

  • Busque palabras con el mismo prefijo. Por ejemplo, en el formulario de consulta escriba key* para encontrar key, keying, keyhole, keyboard, y así sucesivamente.

  • Busque todas las formas de una palabra. Por ejemplo, en el formulario escriba sink** para encontrar sink, sinking, sank y sunk.

  • Busque con la palabra clave NEAR, en lugar de AND, para palabras cercanas entre sí. Por ejemplo, ambas consultas, system and manager y system near manager, buscan las palabras system y manager en la misma página. Pero con NEAR, las páginas devueltas se ordenan por proximidad: cuanto más cerca estén las palabras, mayor será el rango de esa página.

  • Use paréntesis para aclarar la precedencia en una consulta que use operadores booleanos. Por ejemplo, la consulta (dog or cat) and fight encuentra documentos que contienen la palabra fight y una de las palabras dog o cat.

  • Ponga comillas alrededor de las palabras clave si desea que Index Server las trate literalmente. Por ejemplo, si escribe la siguiente consulta:

    "system near manager"

    Index Server buscará literalmente la frase completa system near manager. Pero si escribe la misma consulta sin las comillas:

    system near manager

    Index Server busca en todos los documentos las palabras system y manager en la misma página. Esta búsqueda también incluiría todas las apariciones de la frase system and manager.

  • Refine sus consultas con las palabras clave AND NOT para excluir cierto texto de su búsqueda. Por ejemplo, si desea encontrar todas las instancias de surfing pero no de surfing the Net, escriba la siguiente consulta:

    surfing AND NOT the Net

Estas sugerencias le ayudarán a empezar, pero para consultas más complejas y más ejemplos, consulte la página de Lenguaje de consulta.
