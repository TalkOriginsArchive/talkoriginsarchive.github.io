Conseils de recherche

[Ce qui suit est adapté des instructions sur le langage de requête du Microsoft Index Server]

Au plus simple, une requête peut être un mot ou une phrase. Mais avec les conseils de cette page, vous pouvez élargir le périmètre de votre requête pour obtenir des résultats plus complets. Pour des requêtes plus complexes et davantage d’exemples, consultez la page Langage de requête.

  • Recherchez plus d’un mot. Utilisez les opérateurs booléens AND et OR pour connecter plusieurs mots dans votre requête. Par exemple, tapez dog and cat pour ne trouver que les documents contenant à la fois le mot dog et le mot cat. Tapez dog or cat pour trouver tous les documents contenant l’un ou l’autre des mots. Ne tapez pas dog cat sauf si vous cherchez des occurrences où ces deux mots apparaissent directement côte à côte.

  • Recherchez des mots avec le même préfixe. Par exemple, dans votre formulaire de requête, tapez key* pour trouver key, keying, keyhole, keyboard, et ainsi de suite.

  • Recherchez toutes les formes d’un mot. Par exemple, dans le formulaire tapez sink** pour trouver sink, sinking, sank et sunk.

  • Recherchez avec le mot-clé NEAR, plutôt qu’avec AND, pour des mots proches l’un de l’autre. Par exemple, ces deux requêtes, system and manager et system near manager, recherchent les mots system et manager sur la même page. Mais avec NEAR, les pages retournées sont classées par proximité : plus les mots sont proches l’un de l’autre, plus le classement de cette page est élevé.

  • Utilisez des parenthèses pour clarifier la priorité dans une requête qui utilise des opérateurs booléens. Par exemple, la requête (dog or cat) and fight trouve des documents contenant le mot fight et l’un ou l’autre des mots dog ou cat.

  • Mettez des guillemets autour des mots-clés si vous voulez qu’Index Server les prenne littéralement. Par exemple, si vous tapez la requête suivante :

    "system near manager"

    Index Server cherchera littéralement la phrase complète system near manager. Mais si vous tapez la même requête sans les guillemets :

    system near manager

    Index Server recherche dans tous les documents les mots system et manager sur la même page. Cette recherche inclura également toutes les occurrences de la phrase system and manager.

  • Affinez vos requêtes avec les mots-clés AND NOT pour exclure certains textes de votre recherche. Par exemple, si vous voulez trouver tous les cas de surfing mais pas surfing the Net, écrivez la requête suivante :

    surfing AND NOT the Net

Ces conseils vous permettront de bien démarrer, mais pour des requêtes plus complexes et davantage d’exemples, consultez la page Langage de requête.
